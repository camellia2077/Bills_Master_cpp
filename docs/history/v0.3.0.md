## [v0.3.0] - 2026-02-01
### Summary
- 统一解析/验证/导入流程的领域模型，补齐 ingest 与 convert 能力。
- 引入端口-适配器分层的初步结构，为后续解耦 IO/DB 打基础。

### Refactor
- 统一领域模型：以 ParsedBill/Transaction 作为主数据结构，替换中间 ParentItem/SubItem。
- 新增 JSON 序列化层：集中处理领域模型 ↔ JSON 的读写与持久化。
- 重构验证流程：文本结构校验后转换为领域模型，再进行内容校验（不再依赖 JSON 校验）。
- 新增 convert 命令（-c），默认将文本转换为 JSON 文件；modify 作为兼容别名保留。
- import 流程改为从 JSON 反序列化为领域模型，再由 DB 插入层消费。
- 增加 ingest 命令：支持“全流程插入”，可选落地 JSON。
- 将 reprocessing 目录更名为 conversion，并拆出端口/适配器（读取、序列化、DB）以隔离 IO。
- 配置解析下沉到适配器层：BillConfig/Config 保持纯结构，conversion 不再直接处理 JSON。
- 应用层引入 WorkflowUseCase，控制器仅负责装配与调用，统一用例边界。
- 新增 ConfigProvider 端口与 JSON 适配器，应用层不再直接依赖配置 JSON。
- 引入统一的 Result/Error 模型，集中处理错误返回与消息格式化。
- 用例目录迁移至 application/use_cases，开始稳定分层命名。

### Date Format Migration (ISO)
- 账单日期规范统一迁移为 ISO 月格式：`YYYY-MM`（例如 `2026-01`）。
- 文本账单头 `date:` 输入改为 `YYYY-M`（例如 `date:2026-1`），解析后统一标准化为 `YYYY-MM`。
- JSON 中 `date` 字段仅接受并输出 `YYYY-MM`，不再兼容 `YYYYMM`。
- SQLite `bills.bill_date` 改为存储 `YYYY-MM`，并与 `year`/`month` 字段保持一致。
- CLI 查询与导出月参数改为 `YYYY-MM`，日期范围参数改为 `YYYY-MM YYYY-MM`。
- 移除旧格式兼容路径：`YYYYMM` 作为输入/查询/导出参数不再被接受。
