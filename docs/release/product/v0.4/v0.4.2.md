## [v0.4.2] - 2026-03-10

### Summary
- 收口 `bills_core / bills_io / bills_cli / bills_android` 的共享能力边界，补齐 config 分发、notices 聚合、record template foundation 与 ISO period 统一语义。
- 统一程序内运行目录到 `dist/...`，清理根目录 `output/` 的默认行为，并同步整理测试、构建与发布文档结构。
- Android 客户端进入独立版本线，当前表现层版本为 `0.1.0`，同时保留并展示 `bills_core` 内核版本。

### Added
- 新增 `tools/flows/distribute_configs.py` 与共享 config 分发支持，统一生成 `dist/config/windows` 与 `dist/config/android`。
- 新增 notices 分层清单与聚合生成流程，输出 `dist/notices/windows` 与 `dist/notices/android`，并为 CLI/Android 提供直接查看入口。
- 新增 `bills_core` record template foundation：`template_generate`、`record_preview`、`config_inspect`、`list_periods` 四组共享能力，同时补齐 ABI 命令层。
- 新增 Android `Data / Record / Report / Config` 主界面与 `About` 开源许可入口，支持主题配色、TOML 编辑、record 模板与 period 扫描。
- 新增 product/core/clients 三层发布历史目录，开始区分共享层与客户端历史。

### Changed & Refactored
- 重构程序默认运行目录，CLI 手工运行改为使用 `dist/runtime/bills_tracer/workspace/...`，测试运行与构建产物继续留在 `dist/tests/...` 与 `dist/cmake/...`。
- 重构 SQLite 与查询 period 语义，year 统一为 `YYYY`，month 统一为 `YYYY-MM`，并在 core / CLI / Android 入口保持一致。
- 重构 CMake 共享模块位置，真正跨模块复用的脚本统一落到 `cmake/modules/`，app 私有 CMake 逻辑继续留在各自 `apps/.../cmake/`。
- 重构测试目录与 Python flow 目录，统一到 `tests/suites/...` 与 `tools/flows/...` / `tools/scripts/...` 新结构。
- 重构发布历史目录，跨端产品历史归 `docs/release/product/`，内核能力历史归 `docs/release/core/`，客户端历史归 `docs/release/clients/...`。

### Fixed
- 修复 Android notices metadata 导出任务的 Gradle configuration cache 告警。
- 修复 Android Config 页对 draft TOML 的持久化时机，只有用户点击 `Modify` 后才写回。
- 修复 Android record 已存在日期来源依赖文件名的问题，改为扫描 TXT 内容里的 `date:YYYY-MM`。
- 修复 core / CLI / Android 在 period、文本标准化、查询输入与 config runtime 装配上的分叉行为。
- 修复仓库默认产物落在根目录和本地 IDE 文件易误入版本库的问题，补齐 `dist/runtime/`、`.idea/`、`local.properties` 等忽略规则。
