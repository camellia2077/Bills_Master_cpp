# Android 0.1.0

[Summary]
这是 `bills_android` 的 `0.1.0` 起始版本说明。本次整理基于当前仓库的 Android 工作区改动与最近的 toolchain/config/notices 演进，重点是把手机端体验收口到 `Data / Record / Report / Config` 四个主 tab，并让 Android 直接复用 `bills_core` 的查询、record template、period 扫描与 notices/config 分发能力。

[Added]
- `Data`: 新增面向 Android 私有 workspace 的导入入口，可直接导入 bundled sample，并支持一键清空当前 SQLite 数据库。
- `Record`: 新增基于 `bills_core template_generate` 的 TXT 模板打开能力，支持“新建当前月”和手动输入 `YYYY-MM` 打开模板。
- `Record`: 新增基于 `list_periods` 的已存在日期选择器，年份框只显示已有年份，月份框只显示该年份下已有月份。
- `Record`: 新增 record draft 的 dry-run 预览能力，直接复用 `record_preview`，在保存前就能看到 period、交易数、收支和错误信息。
- `Report`: 新增 Android 侧固定 year / month 查询入口，查询结果可直接显示 core 生成的 Markdown 报表，并可保留原始 JSON 数据。
- `Config`: 新增 Android 构建前 config 分发接入，运行时打包并展示 `validator_config.toml`、`modifier_config.toml`、`export_formats.toml` 三份实际 bundled TOML。
- `Config`: 新增 TOML 下拉切换与可编辑文本框，只有点击 `Modify` 后才持久保存，未确认的修改不会落盘。
- `Config`: 新增 `Notices` 查看入口合并到 `About` 子页，默认展示 `NOTICE.md`，按钮可切到 `notices.json` 原文。
- `Config`: 新增主题配置，支持 Material 3 风格的 `Ember / Harbor / Grove / Canyon` 四套配色，以及 `Follow system / Light / Dark` 三种模式。
- `Config`: 新增版本信息展示，底部明确区分 `core` 内核版本与 `android` 表现层版本。

[Changed & Refactored]
- `Data`: Android 的 sample、config、notices 与 records 统一走私有 materialized workspace，不再依赖硬编码展示文案。
- `Record`: 记录日期来源统一改为扫描 `records/` 下 TXT 文件内容里的 `date:YYYY-MM`，不再依赖文件名推断存在 period。
- `Record`: Record 页直接对接 core 的 `template_generate / record_preview / list_periods`，Android 自身不再维护一套模板拼接和 period 解析逻辑。
- `Report`: 查询输入语义与 core 对齐，只收 ISO period，年份为 `YYYY`，月份为 `YYYY-MM`。
- `Config`: Config 页结构收口为 `TOML / Theme / About` 三个子页，旧的 Notice tab 已并入 Config。
- `Config`: Android 表现层版本号从当前发版起独立记为 `0.1.0`，与 `bills_core` 的内核版本解耦展示。
- `Config`: 主题设置持久化改为 `DataStore`，重新打开应用后保留已应用的颜色与明暗模式选择。
- `Build`: Android 构建前会自动生成分发后的 config 与 notices 资产，再同步进 APK assets，保证 UI 看到的内容与实际打包内容一致。

[Fixed]
- `Record`: 修复了重新进入 Config 后 draft 被误持久化的风险，现在只有显式点击 `Modify` 才会写回 runtime TOML。
- `Record`: 修复了 record 已存在日期选择只能靠文件路径判断的问题，现已改为读取 TXT 内容中的 `date:YYYY-MM`。
- `Report`: 修复了 Android 查询结果只能依赖原始 JSON 阅读的问题，现在默认可直接查看 core 输出的 Markdown。
- `Config`: 修复了 Android 端无法明确区分内核版本与表现层版本的问题，现在底部会分别显示 `core` 与 `android` 两条版本信息。
- `Build`: 修复并清理了 Android notices metadata 导出任务的 configuration cache 兼容性告警，Gradle 配置缓存可正常复用。
