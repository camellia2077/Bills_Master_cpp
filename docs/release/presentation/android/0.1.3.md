## [v0.1.3] - 2026-03-27

### 新增功能 (Added)
* 新增 `Save Record` 的端到端提交流程，`Editor` 现已支持对草稿先校验、再写入 canonical TXT，并在成功后同步 SQLite。
* 新增 `Import TXT from directory` 的单入口提交流程，导入时会同时复制到 `records/` 并同步 SQLite，使 `Query` 与 `Editor` 可立即看到新月份。
* 新增数据库与 canonical TXT 不同步时的显式错误提示，`Editor` 打开缺失源文件的月份时会提示重新导入或重新同步。
* 新增 `Query` 结果区的 `Structured`、`Text`、`Chart` 三态切换，年查询现已支持月度收支柱状图，月查询现已支持支出分类饼图。
* 新增饼图扇区点击后的分类详情卡，用户现可直接查看所选类别的金额与占比。

### 技术改进/重构 (Changed/Refactor)
* 重构 `apps/bills_android/src/main/java/com/billstracer/android/data/services/DefaultWorkspaceService.kt`，将宿主编排与 `Uri/DocumentFile` 读取、临时 staging、native JSON 结果解析拆分到独立协作者，收紧 `WorkspaceService` 的职责边界。
* 重构 `apps/bills_android/src/main/java/com/billstracer/android/app/navigation/AppSessionBus.kt`，将全局 session 状态与 workspace 数据变更通知拆分；`Editor` 与 `Query` 改为通过独立的 `WorkspaceDataChangeBus` 刷新 period 列表。
* 重构 `apps/bills_android/src/main/java/com/billstracer/android/features/editor/EditorViewModel.kt` 与 `apps/bills_android/src/main/java/com/billstracer/android/features/query/QueryViewModel.kt` 的 period 选择逻辑，抽出共享的 `apps/bills_android/src/main/java/com/billstracer/android/features/common/PeriodSelection.kt`，统一年份和月份默认选择规则。
* 重构 `apps/bills_android/src/main/java/com/billstracer/android/features/settings/` 与 `apps/bills_android/src/main/java/com/billstracer/android/platform/` 的大文件结构，将 `SettingsBlocks.kt` 拆为按设置域分离的 block 文件，并将 `MarkdownText.kt` 拆为渲染层、Markdown model 与 parser。
* 重构 `Query` 图表数据链路，将 `grouped_bar` 与 `pie` 所需的图表数据和固定 HEX 颜色统一下沉到 `libs/io` 注入的 `standard_report.extensions.chart_data`，`Android` 改为直接解析并渲染这份 JSON。

### 修复 (Fixed)
* 修复 `Editor` 之前可能出现“TXT 已改但数据库未同步”的中间态，保存失败时现会保留 draft 并回滚本次 persisted TXT 改动。
* 修复 `Workspace` 导入 TXT 后 `Editor` 和 `Query` 需要额外二次导入或手动刷新才能看到月份的问题，导入成功后两处列表现已直接与 SQLite 保持一致。
* 修复 `Editor` 在 tab 切换后文本区域点击会混动回顶部的问题，移除了切 tab 自动刷新引起的重建链路，并隔离了页面外层滚动与文本框内部滚动。
* 修复 `Query` 之前仅能以文本阅读结果的问题，结果区现可按可用数据自动切换到结构化视图或图表视图，并移除用户可见的 `Raw JSON` 入口。
* 修复 `Query` 图表此前由本地主题颜色决定导致类别颜色不稳定的问题，现已改为同一分类跨查询保持固定颜色，并兼容旧版无颜色字段的 JSON 回退渲染。
