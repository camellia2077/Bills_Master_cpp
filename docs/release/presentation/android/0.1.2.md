## [v0.1.2] - 2026-03-22

### 新增功能 (Added)
* 新增 `Workspace` 页 `Data` 卡片与 `Import TXT into database` 按钮，支持将 Android 私有 `records/` 下现有 TXT 批量导入 SQLite，且该入口在 `release` 构建中保留。
* 新增 `Query` 页基于数据库已导入 period 的年份/月份下拉选择，`Year Query` 与 `Month Query` 不再依赖手动数字输入。
* 新增 `Query` 年报的原生 structured preview，支持基于 `standardReportJson` 展示年度汇总与按月明细，并保留 `Markdown / Raw JSON` 切换。
* 新增 `MarkdownText` 的 Markdown 表格渲染支持，使年度 Markdown 报告中的月度汇总表可在 Android 端直接预览。

### 技术改进/重构 (Changed/Refactor)
* 重构 `apps/bills_android/src/main/java/com/billstracer/android/data/runtime/AssetBundleManager.kt` 的 bundled sample materialize 逻辑，sample TXT 改为在工作区初始化时直接落到 `records/`，`Import bundled sample` 仅负责导入数据库。
* 重构 `apps/bills_android/src/main/java/com/billstracer/android/features/workspace/WorkspaceViewModel.kt` 与 `WorkspaceScreen.kt` 的导入状态管理，拆分 bundled sample 导入结果与 `records/` 数据导入结果，避免不同卡片互相覆盖状态。
* 重构 `apps/bills_android/src/main/java/com/billstracer/android/features/query/` 的标准报表解析结构，抽出共享的 `standardReportJson` 解析辅助，并补齐 yearly report 的 Compose 展示链路。

### 修复 (Fixed)
* 修复 `Workspace` 页导入 bundled sample 后 `Editor` 无法打开对应 TXT 的问题，sample 记录现在会按 `records/YYYY/YYYY-MM.txt` 参与工作区初始化。
* 修复 `Query` 页年报 Markdown 在 Android 端只能按纯文本阅读的问题，年报表格现已可按 Markdown 表格渲染。
* 修复 `Query` 页 year query 默认只落到 Markdown 视图的问题，年度查询成功后现已默认进入原生 structured preview。
