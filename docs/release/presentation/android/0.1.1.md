## [v0.1.1] - 2026-03-11

### 新增功能 (Added)
* 新增 `Record` 页手动 period 的 `Year - Month` 双输入框交互，支持固定 `-` 分隔符、数字过滤、月份自动补零与年份输入完成后自动跳转到月份框。
* 新增 `Report` 页可编辑日期查询输入，支持直接输入 `YYYY` 年查询和 `YYYY-MM` 月查询，不再只依赖 bundled sample 的固定日期。
* 新增 `Report` 月报结果的轻量原生展示，优先基于 `standardReportJson` 渲染 Compose 卡片，并保留与 Markdown 视图之间的切换入口。
* 新增 `Data` 页 `Clear all TXT files` 按钮，可直接清空 Android 私有 `records/` 下的全部 `.txt` 文件并同步刷新 `Record` 页状态。
* 新增 `Config` 页主题模式与品牌色选择器，支持预览 Material 3 配色并通过 `DataStore` 持久化主题草稿。

### 技术改进/重构 (Changed/Refactor)
* 重构 `apps/bills_android/src/main/java/com/billstracer/android/ui/` 为 `app`、`common`、`config`、`data`、`record`、`report` 子目录，统一 `Record` 与 `Report` 的日期输入、分组卡片与状态展示组件。
* 重构 `apps/bills_android/src/main/java/com/billstracer/android/ui/BillsViewModel.kt`、`apps/bills_android/src/main/java/com/billstracer/android/data/BillsRepository.kt` 与 `apps/bills_android/src/main/cpp/android_bridge.cpp` 的查询链路，Android 查询改为向 core/native 显式传入 `isoYear` 或 `isoMonth`，并接入 `list_periods`、`coreVersion` 等结构化结果。
* 重构 `Config / About` 中的 Markdown 展示方式，Android 侧改为复用轻量 Compose Markdown 渲染器，而不是把 Markdown 直接按纯文本等宽展示。
* 重构 `Data` 页构建分层，`DEBUG` 构建保留 bundled sample 导入与 `Export TXT and configs` 工具，`release` 构建仅保留私有工作区清理操作。
* 重构 `apps/bills_android/build.gradle.kts` 的 `release` 打包配置，启用 `R8`、`shrinkResources` 与 `arm64-v8a` 单 ABI 打包，并将底部导航图标从 `material-icons-extended` 切换为本地 `drawable` vector 资源。
* 重构 Android 表现层版本线到 `0.1.1`，与 `bills_core` 内核版本继续独立展示与维护。
* 重构 `Record` 页 period 入口为单一流程，移除单独的“当前月”按钮，改为默认填充当前月的 `Year - Month` 输入框加单个 `Open Or Create Period` 按钮。
* 重构 `HeroHeader` 与 `Data / Record` 页面结构，移除顶部重复的 tab 名称胶囊和 period/sample 统计胶囊，页面改为直接展示操作区与状态文本。
* 重构 `Report` 页为 `Year query` 与 `Month query` 两个分组容器，并统一使用 `MaterialTheme.colorScheme.surfaceContainer` 作为功能块背景，使深色模式与 Monet 动态配色自动适配。
* 重构 `Record` 页为 `Open Saved TXT` 与 `Open Or Create Period` 两个分组容器，沿用与 `Report` 相同的 `surfaceContainer` 分组样式，统一查询/编辑入口的视觉结构。

### 修复 (Fixed)
* 修复 `Record` 页手动 period 输入过宽、`YYYY-MM` 一体输入难以校验的问题，现已拆分为更紧凑的 `Year` / `Month` 输入框。
* 修复 `Report` 页查询入口写死日期的问题，现已支持按用户输入查询指定年份或月份。
* 修复 `Report` 页查询时顶部全局进度条干扰阅读的问题，查询结果刷新时不再显示该进度条。
* 修复 Android Markdown 阅读体验偏生硬的问题，月报与 notices 等内容现在可以按更接近文档预览的样式展示。
* 修复 `release` APK 体积异常膨胀的问题，移除全量图标依赖并启用代码/资源收缩后，正式包不再携带不必要的 Compose 图标与未收缩字节码。
* 修复 `Record` 页“当前月”和“手动日期”两个入口语义重复的问题，现已合并为同一组默认带当前月的 period 输入。
* 修复 `Data` 与 `Record` 页顶部视觉噪音偏多的问题，移除不必要的圆角统计容器与辅助说明，界面更聚焦于主操作。
* 修复 `Report` 与 `Record` 页操作块层级不明显的问题，相关功能现已按容器分组显示，用户更容易区分“打开已有内容”和“按日期查询/创建”的不同区域。
