# CLI 0.1.0

[Summary]
这是 `bills_cli` 的 `0.1.0` 起始版本说明。本次版本以提交 `8abd341`（`refactor: rebuild cli command groups`）为主，结合 `temp/commit_cli_v042.txt` 的整理结果，核心变化是将原先手写的 flag/dispatcher/controller 式 CLI 重建为基于 `CLI11` 的命令树解析实现，并把命令面统一收口到 `workspace / report / template / config / meta` 五组 typed command。

[Breaking Changes]
- 移除旧的 flag 风格入口、手写 command dispatcher、command classes 与 `windows/presentation/cli/controllers/*` 控制器链路，旧命令脚本需要按新的子命令语法迁移。
- 移除 `help` 子命令、`--version`、`--notices` 与 `--notices-json` 这类旧入口，分别改为 `--help`、`meta version`、`meta notices [--json]`。
- 移除 CLI 对外暴露的 export pipeline 选择入口，导出命令面只保留新的 `report export ...` 子命令语义。

[Added]
- 新增 `CLI11` 依赖接入，`apps/bills_cli/cmake/dependencies.cmake` 通过 `FetchContent` 拉取 `CLI11 v2.6.1`，CLI 解析不再依赖仓库内手写分发器。
- 新增 `apps/bills_cli/src/presentation/parsing/cli_parser.cpp` 的 typed request 解析链路，统一把参数解析成 `WorkspaceRequest / ReportRequest / TemplateRequest / ConfigRequest / MetaRequest`。
- 新增 `apps/bills_cli/src/presentation/entry/runtime_context.cpp` 的运行时上下文解析，统一处理 repo root、runtime workspace、默认数据库、JSON cache、导出目录与 notices/config 路径。
- 新增 `apps/bills_cli/src/presentation/features/` 下按能力分组的 handler：`workspace`、`report`、`template`、`config`、`meta`，表现层入口改为 feature-scoped 调度。
- 新增基于 `CLI11` 的 help 文本、示例 footer 与 `<command> --help` 交互，帮助输出改为围绕 typed subcommand 展示。

[Changed & Refactored]
- 将 CLI 可执行入口重构为 `apps/bills_cli/src/presentation/entry/cli_app.cpp`，入口只负责参数收集、request 解析、runtime context 构建和 feature handler 调度。
- 将 CLI 命令面重构为五组顶层命令：
  - `workspace`: `validate / convert / ingest / import-json / export-bundle / import-bundle`
  - `report`: `show year|month` 与 `export year|month|range|all-months|all-years|all`
  - `template`: `generate / preview / list-periods`
  - `config`: `inspect / formats`
  - `meta`: `version / notices`
- 将模板、报表和工作区命令的入参改为更明确的 positional/option 组合，例如 `template generate --period <YYYY-MM>`、`report export range <YYYY-MM> <YYYY-MM>`，不再依赖旧 flag 混合模式。
- 将 CLI 表现层源码主线从 `apps/bills_cli/src/windows/presentation/cli/` 收口到 `apps/bills_cli/src/presentation/`，旧的 `command_handler`、`usage_help`、`controllers/workflow`、`controllers/export` 等实现整体下线。
- 更新 artifact/task 脚本和模块文档到新的命令语法，`tests/framework/internal/tasks.py` 已改为走新的 `workspace / report / template / config / meta` 命令面。

[Fixed]
- 修复旧手写 CLI 帮助文案和真实命令面容易漂移的问题，帮助输出现在由 `CLI11` 命令树直接生成。
- 修复旧 CLI 参数分发职责过重、入口与控制器链路耦合过深的问题，参数解析、运行时路径解析和功能执行现在分离得更清楚。
- 修复旧 CLI 入口对 notices/version 等元信息命令依赖特例 flag 的问题，相关能力统一迁移到 `meta` 命令组。
- 修复旧 CLI 表现层扩展新命令时需要同时改 dispatcher、command class、controller 的高维护成本问题，现在新增命令主要落在 parser + typed request + handler 三处。
