## [v0.3.2] - 2026-03-09

### 新增功能 (Added)
* 新增 `typ` 原生报表渲染能力，`libs/bills_core` 现在可直接从 `StandardReport` 生成 `.typ` 文本。
* 新增 `StandardReport` 统一 renderer registry，支持按编译结果暴露可用格式列表。
* 新增 `libs/bills_core/CMakeLists.txt` 格式编译开关：`BILLS_CORE_ENABLE_RENDERER_JSON`、`BILLS_CORE_ENABLE_RENDERER_MD`、`BILLS_CORE_ENABLE_RENDERER_RST`、`BILLS_CORE_ENABLE_RENDERER_TEX`、`BILLS_CORE_ENABLE_RENDERER_TYP`。
* 新增共享文本规范化入口 `libs/bills_core/src/common/text_normalizer.*`，统一处理 `UTF-8 BOM`、`CRLF/CR` 和非法 `UTF-8`。
* 新增 `record template foundation`：`bills_core` 现在提供 `template_generate`、`record_preview`、`config_inspect`、`list_periods` 四组能力，供 CLI、Android 和 ABI 复用。
* 新增有序模板布局解析模型，模板输出现在可保留 `validator_config.toml` 的 `categories / parent_item / description / sub_items` 原始顺序，用于 TXT 模板生成与配置检查。

### 技术改进/重构 (Changed/Refactor)
* 重构报表导出链路为 `SQLite adapter -> Monthly/Yearly report data -> StandardReport -> format renderer -> exported text`，`ReportExportService` 不再直接依赖月报/年报 formatter provider。
* 收口 `bills_core` 为格式编译开关的唯一归属地，Windows CLI 默认编译全格式 `json/md/rst/tex/typ`，Android native 只编译 `json/md`。
* 统一 `apps/bills_cli` 到单一 `libs/bills_core` 目标，不再维护一份 CLI 私有 core 源文件清单和旧格式开关。
* 收口 Android 查询输出，JNI 在保留结构化 `standard_report` JSON 的同时新增 `report_markdown`，用于 Compose 侧直接渲染 Markdown。
* 迁移 Android 资产装配为构建期从根目录 `testdata/bills` 与 `config/` 同步到 assets，Gradle 产物回归模块默认 `apps/bills_android/build`，CMake staging 回归模块默认 `apps/bills_android/.cxx`。
* 更新报表快照基线与冻结脚本，基线样例切换到 `2025` 数据集，并将 `Typst_bills/...` 纳入 `tests/baseline/report_snapshots` 对比范围。
* 将 `record_template` 逻辑拆成 `ordered_template_layout_loader`、`template_render_support`、`period_support`、`file_support`，`RecordTemplateService` 收口为编排层，不再同时承担 TOML 解析、period 校验、模板渲染和文件枚举四类职责。
* 新增共享 `RuntimeConfigLoader`，统一 ABI 与 record-template 流程对 `validator_config.toml` / `modifier_config.toml` 的文件读取与校验装配。
* 新增共享 `common/iso_period.*`，统一 `YYYY` / `YYYY-MM` 的解析、格式化与 `date:YYYY-MM` 头提取逻辑，查询、validator、parser 与 record-template 现在共用同一套 ISO period 规则。
* 将 record-template 错误模型从字符串分类升级为强类型 `RecordTemplateErrorCategory`，ABI 与 CLI 的错误映射改为基于同一份 typed error 输出。

### 修复 (Fixed)
* 修复 Windows、Android 与 ABI 对同一份账单 `.txt` 在换行和 `UTF-8 BOM` 处理上不一致的问题，统一在 parser 前走同一份文本规范化逻辑。
* 修复 repo-owned 文本读取器依赖平台文本模式的问题，账单文本读取统一改为二进制读取，避免 `std::ifstream` 文本模式在 Windows 下隐式折叠换行。
* 修复 ABI `validate`、`convert`、`ingest` 在 normalizer 或业务失败时错误信息不稳定的问题，现有 `files[].error` 字段可稳定返回详细失败原因。
* 修复 Android 查询展示只能依赖原始 JSON 的问题，查询结果现在可直接显示 core 生成的 Markdown 报表。
* 修复 core、ABI、CLI 各自重复实现 period 校验与 TOML runtime config 装配的问题，避免相同输入在不同入口出现校验分叉。
* 修复 record-template 重构过程中 `TxtStructureVerifier` 缺失 `<regex>` 依赖导致的构建失败，`remark:` 行校验现已恢复稳定编译。

### 弃用/删除 (Deprecated/Removed)
* 删除 `bills_io` 里的旧 formatter provider 装配入口：`CreateMonthReportFormatterProvider()` 与 `CreateYearlyReportFormatterProvider()`。
* 删除旧的月报/年报 generator 与 formatter provider 端口实现，报表导出统一改走 `StandardReport` renderer 路径。
* 删除 CLI 私有旧格式开关 `ENABLE_FMT_MD`、`ENABLE_FMT_RST`、`ENABLE_FMT_TEX` 以及 `BILLS_FORMAT_COMPILE_DEFINITIONS`，格式可用性不再由使用方自行猜测。
