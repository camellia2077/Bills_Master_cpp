## [v0.4.1] - 2026-03-06

### 新增功能 (Added)
* 新增 `libs/bills_core/src/config_loading/config_bundle_validation.*`，统一校验 `validator_config.toml`、`modifier_config.toml` 与 `export_formats.toml`，并输出带 `code`、`field_path`、`line`、`column` 的结构化 `ValidationIssue` 列表。
* 新增 `libs/bills_core/src/record_template/import_preflight_service.*`，在正式导入前同时检查配置合法性、记录批次重复 period、文件名 period 与内容 period 不一致，以及工作区/数据库 period 冲突。
* 新增 `validate_config_bundle`、`validate_record_batch`、`preflight_import` 等 ABI 命令，统一为 CLI/Android/native 调用方返回结构化 JSON 结果与跨文件校验详情。

### 技术改进/重构 (Changed/Refactor)
* 重构 `libs/bills_core/src/record_template/record_template_service.*`，将记录预览、批量校验与 `list_periods` 收敛为统一服务入口，并同时支持 `config_dir` 与显式 `validator` / `modifier` 文件路径两种调用口径。
* 重构 `libs/bills_core/src/reports/standard_json/standard_report_json_serializer.*` 与 `libs/bills_core/src/reports/core/standard_json_*_renderer.cpp`，统一以 canonical `StandardReport` JSON 作为 `md`、`rst`、`tex`、`typ` 渲染的共享中间层。
* 重构 `libs/bills_core/src/reports/core/standard_report_render_support.hpp` 中的 period 标签与 remark 格式化逻辑，统一 `YYYY-MM` 标题、月份解析与空 remark 占位行为。
* 重构 `apps/bills_cli/src/windows/presentation/cli/controllers/app_controller.cpp` 的运行时工作区与 `export_formats.toml` 装配流程，CLI 导出能力改为依据校验后的 `enabled_formats` 与当前构建可用格式共同判定。
* 重构 `tests/baseline/report_snapshots/`、`tools/verify/check_report_snapshots.py` 与 `tools/verify/freeze_report_snapshots.py` 的基线策略，新增 `standard_report` golden，并将 `md`、`json`、`rst`、`tex`、`typ` 输出统一纳入快照回归检查。

### 修复 (Fixed)
* 修复 `export_formats.toml` 中未知格式、重复格式或当前构建未启用格式无法被准确识别的问题，现会返回明确的 `business` / `schema` 校验错误。
* 修复记录批量预检中 period 冲突信息不足的问题，现会分别报告批次内重复 period、文件名/内容 period 不一致，以及工作区/数据库冲突。
* 修复多种报表 renderer 在月份标签与标准 JSON 结构上的口径漂移问题，`md`、`rst`、`tex`、`typ` 与 canonical `standard_report` 快照现已统一。
