## [v0.4.2] - 2026-03-25

### 新增功能 (Added)
* 新增 `libs/bills_core/src/config/`、`libs/bills_core/src/ingest/`、`libs/bills_core/src/query/`、`libs/bills_core/src/reporting/` 与 `libs/bills_core/src/common/source_document.hpp`，将 `bills_core` 主线能力收口为纯数据配置校验、导入、查询与标准报表渲染入口。
* 新增 `libs/bills_core/src/common/iso_period.*` 与 `libs/bills_core/src/common/text_normalizer.*`，统一 `YYYY` / `YYYY-MM` period 解析、格式化与文本标准化语义。
* 新增 `libs/bills_core/src/record_template/` 下的 `record_template_service.*`、`record_template_types.hpp`、`period_support.*`、`template_render_support.*`、`ordered_template_layout_loader.*` 与 `import_preflight_service.*`，补齐 `template_generate`、`record_preview`、`list_periods` 与导入预检共享能力。
* 新增 `libs/bills_io/src/bills_io/adapters/config/config_document_parser.*`、`adapters/io/source_document_io.*`、`adapters/io/json_bill_document_io.*`、`adapters/reports/report_export_service.*` 与 `host_flow_support.*`，将宿主侧配置读取、源文档输入、JSON 缓存与报表导出统一沉淀到 `bills_io`。
* 新增 `libs/bills_io/src/bills_io/adapters/io/zip_archive_io.*` 与 `libs/bills_core/src/ingest/workflow_validation_issue_support.hpp`，补齐 parse bundle ZIP 导出导入与导入问题建模支撑。

### 技术改进/重构 (Changed/Refactor)
* 重构 `libs/bills_core/src/abi/bills_core_abi.cpp` 与相关 ABI 服务链路，`validate_config_bundle`、`template_generate`、`validate_record_batch`、`preflight_import`、`validate`、`convert`、`ingest`、`import`、`query` 统一改为接收结构化文档或纯数据请求，不再依赖路径驱动协议。
* 重构 `libs/bills_core/src/billing/conversion/` 到 `libs/bills_core/src/ingest/`，并按 `convert`、`pipeline`、`transform`、`validation`、`json` 重新归并导入主线实现，收紧 `core` 对 IO 细节的依赖边界。
* 重构 `libs/bills_core/src/reports/` 到 `libs/bills_core/src/reporting/`，统一 `StandardReport` 组装、renderer registry 与 `json/md/rst/tex/typ` 渲染主线。
* 重构 `libs/bills_core/src/modules/` 的导出面，删除旧 `workflow_use_case`、旧 ABI shared、旧 report exporter 与旧文件型 port 模块，统一切换到 canonical config / ingest / query / reporting / record-template 模块入口。
* 重构 `libs/bills_io` 的配置与宿主接线方式，CLI/Android 共享装配统一经由 `host_flow_support` 进入 `bills_core` 纯数据服务，`bills_io` 自身只承接配置文档解析、源文档读取、sqlite 访问、导出落地与 bundle IO。
* 重构 `libs/bills_io/src/bills_io/adapters/reports/report_export_service.*` 与相关报表数据链路，导出能力改为围绕标准报表渲染器和 ISO period 统一语义工作，不再保留旧 formatter provider 装配路径。
* 重构 `libs/bills_core` 与 `libs/bills_io` 的 period 约定，year 固定为 `YYYY`、month 固定为 `YYYY-MM`，查询、模板、导出与 SQLite month 列举统一采用同一口径。

### 修复 (Fixed)
* 修复 `libs/bills_core` 活跃源码中混入 `toml::`、`sqlite3`、`<fstream>`、目录遍历与导出写盘职责的问题，`core` 现在只保留业务规则与纯数据编排。
* 修复 record template、导入预检、查询与报表导出之间 period 解析不一致的问题，`iso_period` 与 `text_normalizer` 已成为共享事实来源。
* 修复标准报表快照与多格式渲染结果容易因非业务字段或不同渲染支撑而漂移的问题，标准报表 JSON、renderer registry 与 snapshot 对比链路已对齐。
* 修复 parse bundle 导入失败时 runtime config 或目标 `records/` 可能被部分写入的问题，bundle 宿主流程已补齐失败回滚与结构校验。
* 修复宿主侧 `config inspect`、`report` 与 `workspace` 子命令各自散落装配逻辑的问题，相关共享准备步骤已经收口到 `libs/bills_io/src/bills_io/host_flow_support.*`。

### 弃用/删除 (Deprecated/Removed)
* 删除 `libs/bills_core/src/application/use_cases/workflow_use_case.*`、`libs/bills_core/src/config_loading/`、`libs/bills_core/src/config_validator/`、`libs/bills_core/src/db_insert/`、`libs/bills_core/src/serialization/`、旧 `abi/internal/handlers/` 与旧路径型 `ports/*`，不再保留 legacy workflow、旧配置加载链路与文件型端口兼容层。
* 删除 `libs/bills_core/src/reports/core/report_exporter.*`、旧 `monthly_report` / `yearly_report` generator 实现与旧 formatter/provider 依赖面，报表能力统一归并到 `reporting/` 标准报表渲染主线。
* 删除 `libs/bills_io/src/bills_io/adapters/config/toml_*`、`adapters/io/file_bill_*`、`io_factory.*` 的旧配置/文件型辅助假设以及旧 built-in formatter provider，实现统一迁移到 `config_document_parser`、`source_document_io`、`json_bill_document_io`、`report_export_service`、`zip_archive_io` 与 `host_flow_support`。
