## [v0.4.2] - 2026-03-21

### 新增功能 (Added)
* 新增 `libs/bills_core/src/config/`、`libs/bills_core/src/ingest/`、`libs/bills_core/src/query/` 与 `libs/bills_core/src/reporting/`，将 `bills_core` 的主线能力重组为纯数据配置校验、导入批处理、查询组装与报表渲染服务。
* 新增 `libs/bills_core/src/common/source_document.hpp`，统一使用 `{display_path, text}` 文档批次作为跨层输入，不再在 `core` 内部直接枚举文件或读取文本。
* 新增纯数据 `bills_core` ABI 实现，`validate_config_bundle`、`template_generate`、`validate_record_batch`、`preflight_import`、`validate`、`convert`、`ingest`、`import`、`query` 统一改为接收结构化 JSON 文档而非路径参数。

### 技术改进/重构 (Changed/Refactor)
* 重构 `libs/bills_core/src/billing/conversion/*` 到 `libs/bills_core/src/ingest/`，并按 `pipeline`、`convert`、`transform`、`validation`、`json` 重新归并导入主线叶子实现。
* 重构 `libs/bills_core/src/reports/*` 到 `libs/bills_core/src/reporting/`，并按 `standard_report`、`renderers`、`sorters` 重新归并 `StandardReport`、renderer registry 与排序实现。
* 重构 `apps/bills_cli/src/windows/presentation/cli/controllers/` 与 `apps/bills_android/src/main/cpp/android_bridge.cpp` 的宿主接线方式，CLI/Android 改为统一通过 `libs/bills_io/src/bills_io/host_flow_support.*` 准备配置与文档，再调用 `bills_core` 的纯数据服务。
* 重构 `libs/bills_core/src/record_template/`，模板生成、记录预览、period 列举与 preflight import 统一切换为文档批次输入；`TemplateGenerationRequest`、`RecordPreviewResult`、`ListedPeriodsResult` 等类型去除路径驱动字段，保留展示用字符串。
* 重构 `libs/bills_core/src/modules/`，删除旧 `workflow_use_case`、`config_validator`、文件型 port、旧 ABI shared 与旧 report exporter 模块，统一切换到 `bill.core.config.bundle_service`、`bill.core.ingest.*`、`bill.core.query.query_service`、`bill.core.reporting.*`、`bill.core.record_template.*`、`bill.core.abi.entry` 新模块名。
* 重构 `libs/bills_core/src/ingest/json/bills_json_serializer.*`，账单 JSON 序列化改为纯字符串编解码，不再在 `core` 内部直接读写 `.json` 文件。
* 重构 `tools/verify/check_report_snapshots.py` 与 `tools/verify/report_snapshot_support.py`，项目间 `JSON_bills` 比对统一走 JSON 渲染归一化，忽略 `generated_at_utc` 造成的非业务漂移。
* 重构 `tools/verify/tools/config/import_layer_whitelist.json` 基线，收敛到新 `core/io` 结构下的 quoted include 实际使用面。

### 修复 (Fixed)
* 修复 `bills_core` 中 `config_loading/`、`record_template/file_support.*`、`reports/core/report_exporter.*`、旧 ABI handler 与旧文件型 serializer/provider 混入 IO 细节的问题，`core` 源码不再直接依赖 `toml::`、`sqlite3`、`<fstream>`、`recursive_directory_iterator`、`create_directories`。
* 修复 `report-consistency-gate` 对 `JSON_bills` 产物的误报问题，现已避免 `generated_at_utc` 时间戳导致的跨导出管线伪差异。
* 修复 `module-mode-check` 仍依赖旧模块名与旧 smoke 的问题，模块门禁现已切换到新 canonical 目录对应的导出面并恢复通过。
* 修复 `core-abi` 仍依赖旧路径型命令契约的问题，ABI smoke tests 现已切换到新的纯数据请求模型并恢复通过。

### 弃用/删除 (Deprecated/Removed)
* 删除 `libs/bills_core/src/application/use_cases/workflow_use_case.*`、`libs/bills_core/src/config_loading/`、`libs/bills_core/src/config_validator/`、`libs/bills_core/src/db_insert/`、`libs/bills_core/src/serialization/`、`libs/bills_core/src/reports/`、旧 `abi/internal/handlers/` 与旧路径型 `ports/*`，不再保留向前兼容入口。
* 删除 `libs/bills_io/src/bills_io/adapters/config/toml_*`、`libs/bills_io/src/bills_io/adapters/io/file_bill_*` 与旧 CLI `path_builder.*` 等历史适配器实现，统一改由 `config_document_parser`、`source_document_io`、`json_bill_document_io`、`host_flow_support` 与新的 `report_export_service` 接管。
